## Gap Analysis

### 1. Authentication (`authentication.spec.ts`)

**Happy paths covered:**
- Login with each customer account (parameterized x4)
- Logout after login
- Session persistence after page reload

**Negative scenarios covered:**
- Invalid password for valid username
- Non-existent/too-short username
- Guest credentials with wrong password

**GAPS - Missing negative scenarios:**
- [ ] Empty username submission
- [ ] Empty password submission
- [ ] Empty username AND password submission
- [ ] SQL injection / XSS in username field (security edge case)
- [ ] Login with correct guest username + correct guest password (tests what guest login actually does)
- [ ] Rapid repeated login attempts (rate limiting)
- [ ] Password field shows masked input (UI validation)

**GAPS - Missing edge cases:**
- [ ] Session expiry behavior (if applicable)
- [ ] Multiple tab login/logout synchronization
- [ ] Login form field validation messages disappear on correction
- [ ] Username/password fields have correct maxlength/input constraints

---

### 2. Cart Management (`taqelah-cart-management.spec.ts`)

**Happy paths covered:**
- Add multiple products to cart
- Update quantity (increase)
- Remove an item from cart
- Cart totals update correctly
- Checkout button is visible

**GAPS - Missing happy paths:**
- [ ] Add a single product and verify cart count badge updates
- [ ] Decrease quantity of an item
- [ ] Add the same product twice (verify quantity increments vs duplicate entry)
- [ ] Cart persists across page reload (localStorage persistence)
- [ ] Open and close cart panel multiple times

**GAPS - Missing negative scenarios:**
- [ ] Remove ALL items from cart (empty cart state)
- [ ] Attempt checkout with empty cart
- [ ] Set quantity to 0 (should remove item or show error)
- [ ] Set quantity to very high number (boundary testing)
- [ ] Cart icon badge shows 0 or is hidden when cart is empty

**GAPS - Missing edge cases:**
- [ ] Cart total calculation with different product prices
- [ ] Cart state after logout and re-login (does it persist per user?)
- [ ] Add product, logout, login as different user (cart isolation)

---

### 3. Checkout Flow (`taqelah-checkout-final.spec.ts`)

**Happy paths covered:**
- Authenticated user adds item to cart
- Opens checkout modal
- Applies promo code (conditionally)
- Fills all form fields
- Places order
- Verifies order confirmation (order number, total)
- Returns to homepage via "Continue Shopping"

**GAPS - Missing happy paths:**
- [ ] Checkout with multiple items in cart
- [ ] Checkout WITHOUT promo code (explicit test, not conditional skip)
- [ ] Verify order total matches cart total
- [ ] Verify each promo code type works (SAVE10, WELCOME20, FREESHIP)
- [ ] Checkout with different country selections

**GAPS - Missing negative scenarios:**
- [ ] Submit checkout form with empty required fields (no name, no email, etc.)
- [ ] Submit with invalid email format
- [ ] Submit with invalid phone number format
- [ ] Submit with invalid postal code
- [ ] Apply invalid/expired promo code
- [ ] Apply promo code twice (duplicate application)
- [ ] Remove applied promo code before placing order
- [ ] Place order button disabled until all required fields are filled
- [ ] Close checkout modal and verify cart is preserved

**GAPS - Missing edge cases:**
- [ ] Checkout form field validation messages
- [ ] Promo code case sensitivity
- [ ] Very long values in form fields
- [ ] Special characters in name/address fields
